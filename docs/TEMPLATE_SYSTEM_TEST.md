# 模版系统测试文档

## 测试环境

- **浏览器**: Chrome 88+
- **页面**: 携程 VBK 产品详情页 (`/vendor/baseInfoMerge`)
- **扩展版本**: 0.1.0+

## 自动化测试

### 运行方法

1. 打开产品详情页
2. 打开浏览器控制台（F12）
3. 复制 `test/template-system-test.js` 内容并粘贴到控制台
4. 运行测试命令

### 测试命令

```javascript
// 运行所有测试
TemplateSystemTest.runAll()

// 运行单个测试
TemplateSystemTest.testSaveFlow()      // 测试保存流程
TemplateSystemTest.testEditFlow()      // 测试编辑流程
TemplateSystemTest.testDeleteFlow()    // 测试删除流程
TemplateSystemTest.testPreviewFlow()   // 测试预览流程
TemplateSystemTest.testDuplicateName() // 测试重名
TemplateSystemTest.testMaxTemplates()  // 测试数量限制
TemplateSystemTest.testApplyFlow()     // 测试应用模版

// 查看当前状态
TemplateSystemTest.status()

// 清理测试模版
TemplateSystemTest.cleanup()
```

## 手动测试用例

### 1. 保存流程测试

#### 测试用例 1.1：基本保存流程

**前置条件**：
- 已打开产品详情页
- 页面表单已填写一些数据

**测试步骤**：
1. 点击页面底部"💾 保存为模版"按钮
2. 对话框弹出，显示默认名称"产品模版 1"
3. 修改名称为"测试模版A"
4. 输入描述"这是第一个测试模版"
5. 点击"💾 保存"按钮

**预期结果**：
- ✅ 对话框顶部显示绿色提示"✅ 模版已保存"
- ✅ 对话框自动关闭
- ✅ 控制台无错误

**验证**：
```javascript
const templates = await TemplateStorage.getAll();
console.log('已保存模版:', templates);
// 应该看到刚保存的模版
```

#### 测试用例 1.2：名称为空

**测试步骤**：
1. 点击"💾 保存为模版"
2. 清空名称输入框
3. 观察保存按钮状态

**预期结果**：
- ✅ 保存按钮变灰，禁用状态
- ✅ 无法点击保存

#### 测试用例 1.3：重名检测

**前置条件**：
- 已存在名为"测试模版A"的模版

**测试步骤**：
1. 点击"💾 保存为模版"
2. 输入名称"测试模版A"
3. 点击"💾 保存"

**预期结果**：
- ✅ 显示红色错误提示："模版名称已存在，请使用不同的名称"
- ✅ 对话框不关闭，允许修改

#### 测试用例 1.4：长名称和长描述

**测试步骤**：
1. 点击"💾 保存为模版"
2. 输入51个字符的名称
3. 输入201个字符的描述

**预期结果**：
- ✅ 名称输入框最多只能输入50个字符
- ✅ 描述输入框最多只能输入200个字符

#### 测试用例 1.5：ESC 键关闭

**测试步骤**：
1. 点击"💾 保存为模版"
2. 按 ESC 键

**预期结果**：
- ✅ 对话框关闭，不保存

#### 测试用例 1.6：点击遮罩关闭

**测试步骤**：
1. 点击"💾 保存为模版"
2. 点击对话框外的黑色遮罩

**预期结果**：
- ✅ 对话框关闭，不保存

### 3. 应用模版测试

#### 测试用例 3.1：从导入面板应用模版

**前置条件**：
- 已保存至少1个模版

**测试步骤**：
1. 点击"📥 导入数据"按钮（国际版页面）
2. 选择"● 使用已保存的模版"单选按钮
3. 从下拉框选择一个模版
4. 观察提示信息
5. 点击"下一步：解析并转换"

**预期结果**：
- ✅ JSON 输入区隐藏
- ✅ 模版选择区显示
- ✅ 下拉框列出所有模版（按更新时间倒序）
- ✅ 选择后显示："📦 28 个字段 | 更新于 2026-06-15"
- ✅ "下一步"按钮可用
- ✅ 点击后进入转换预览步骤
- ✅ 显示所有字段和值

#### 测试用例 3.2：切换数据源

**测试步骤**：
1. 打开导入面板，默认选中"粘贴 JSON 数据"
2. 切换到"使用已保存的模版"
3. 再切换回"粘贴 JSON 数据"

**预期结果**：
- ✅ 切换到模版：JSON 输入区隐藏，模版选择区显示
- ✅ 切换回 JSON：模版选择区隐藏，JSON 输入区显示
- ✅ 按钮状态正确更新

#### 测试用例 3.3：页面类型不匹配

**前置条件**：
- 在国内版页面保存模版（pageType: "product-detail"）
- 当前在国际版页面（pageType: "product-detail"）

**测试步骤**：
1. 选择"使用已保存的模版"
2. 选择国内版页面保存的模版
3. 点击"下一步：解析并转换"

**预期结果**：
- ✅ 如果 pageType 相同，直接进入转换预览
- ✅ 如果 pageType 不同，弹出确认对话框：
  ```
  ⚠️ 页面类型不匹配

  模版创建于：产品详情页
  当前页面是：产品详情页

  可能存在字段不匹配，是否继续？
  ```
- ✅ 点击"取消"停留在步骤1
- ✅ 点击"确定"继续到转换预览

#### 测试用例 3.4：无模版时的状态

**前置条件**：
- 删除所有模版

**测试步骤**：
1. 打开导入面板
2. 选择"使用已保存的模版"

**预期结果**：
- ✅ 下拉框显示"暂无模版"并禁用
- ✅ "下一步"按钮禁用
- ✅ 提示文字："还没有保存的模版，请先在页面底部点击'💾 保存为模版'"

#### 测试用例 3.5：完整导入流程

**测试步骤**：
1. 打开导入面板
2. 选择"使用已保存的模版"
3. 选择一个模版
4. 点击"下一步：解析并转换"
5. 查看转换预览，点击"匹配预览"
6. 查看匹配结果，点击"确认填写"
7. 等待填写完成
8. 查看回读验证结果

**预期结果**：
- ✅ 转换预览：显示所有字段和值
- ✅ 匹配预览：显示匹配/未匹配/无目标字段统计
- ✅ 确认填写：显示进度条和填写结果
- ✅ 回读验证：显示验证结果（成功/失败/跳过）
- ✅ 表单数据正确填入

1. 保存为模版
2. 打开模版管理，点击预览

- ✅ 保存成功，响应时间 < 500ms

- ✅ 特殊字符正确转义，不执行脚本
- ✅ HTML 标签显示为纯文本
- ✅ 无 XSS 风险

## 性能测试

### 性能要求

| 操作 | 预期时间 |
|------|---------|
| 保存模版 | < 500ms |
| 加载模版列表 | < 200ms |
| 应用模版 | < 1s |
| 删除模版 | < 200ms |
| 打开预览 | < 300ms |

### 测试方法

```javascript
// 测试保存性能
console.time('保存模版');
await TemplateManager.createFromCurrentPage('性能测试', '测试');
console.timeEnd('保存模版');

// 测试加载性能
console.time('加载列表');
await TemplateManager.getAllSorted();
console.timeEnd('加载列表');
```
