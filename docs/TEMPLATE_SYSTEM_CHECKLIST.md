## 开发建议

### 推荐开发顺序

1. **Phase 1 → Phase 2**（2-3小时）
   - 先实现存储和业务逻辑，可以在控制台测试
   - 不依赖 UI，可以快速验证核心功能

2. **Phase 3**（2-3小时）
   - 实现保存模版的 UI
   - 可以立即看到效果，提升成就感

3. **Phase 4**（3-4小时）
   - 实现模版管理面板
   - 最复杂的 UI 部分

4. **Phase 5**（1-2小时）
   - 应用模版功能

5. **Phase 6 → Phase 8**（2-3小时）
   - 测试和优化
   - 文档更新

**总预计时间：12-17 小时**

### 调试技巧

在控制台快速测试存储层：
```javascript
// 测试保存
await TemplateStorage.save({
  id: 'tpl_test',
  name: '测试模版',
  description: '测试用',
  pageType: 'product-detail',
  data: { test: 'data' },
  createdAt: new Date().toISOString(),
  updatedAt: new Date().toISOString()
});

// 测试读取
const all = await TemplateStorage.getAll();
console.log('所有模版:', all);

// 测试删除
await TemplateStorage.delete('tpl_test');
```

### 常见陷阱

1. **chrome.storage.local 是异步的**
   - 所有操作都要用 `await`
   - 不要忘记 `async` 关键字

2. **模版数据是嵌套对象**
   - 注意深拷贝 vs 浅拷贝
   - 不要直接修改 storage 返回的对象

3. **UI 状态同步**
   - 保存/删除后要刷新列表
   - 使用事件或回调通知其他组件

4. **时间戳格式**
   - 统一使用 ISO 8601：`new Date().toISOString()`
   - 显示时格式化为本地时间
