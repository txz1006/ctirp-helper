> 版本：v0.3.0 | 创建：2026-06-14 | 更新：2026-06-21
>
> 本文档只记录产品需求与决策，技术实现见 [DEVELOPMENT_GUIDE.md](DEVELOPMENT_GUIDE.md)，项目导航见 [CLAUDE.md](CLAUDE.md)。

## 一、项目背景

将携程国内版 VBK 后台的产品数据迁移到 Trip.com 国际版后台。两个系统同域名 `vbooking.ctrip.com`，表单结构相似但有差异（字段映射、ID 体系不同、中文需翻译）。

此前 API 逆向方案已证可行，但 ID 体系不匹配（城市 ID、路线 ID 不通用），部分字段无法通过纯 API 方式正确写入。本方案在浏览器表单层做导出/导入，绕过后端 ID 问题。

**技术栈：** React 16.14.0 + Ant Design（新旧版本混合）+ SSR(国内) / CSR(国际)

## 二、产品目标

1. 国内版页面**一键导出**当前 Tab 的表单数据为 JSON
2. 国际版页面**一键导入**，自动填入表单
3. 人审核确认后保存

## 三、核心决策记录

> 编号前缀 P = Product decision（产品决策）。架构决策见 [ARCHITECTURE_REFACTOR.md](docs/ARCHITECTURE_REFACTOR.md)（编号前缀 D = Design decision）。

| # | 决策项 | 选择 | 理由 |
|---|--------|------|------|
| P1 | 数据提取 | DOM 值为主，React fiber 为辅 | DOM 覆盖大部分字段；搜索下拉框从 fiber 取显示文本 |
| P2 | 填写策略 | MVP 仅 DOM 模拟 | 先验证覆盖率，API 兜底放到后续迭代 |
| P3 | 翻译方式 | 预置规则 + 大模型（仅自由文本） | 结构化字段用规则，减少 API 调用 |
| P4 | 大模型 Key | 用户自填 | 避免共享 Key 的安全和费用问题 |
| P5 | 插件形态 | Chrome Extension (Manifest V3) | 需注入按钮、浮层 UI、跨页面状态 |
| P6 | 数据传递 | 剪贴板优先，失败降级文件下载 | 数据量小（几 KB） |
| P7 | 操作粒度 | 按 Tab 逐个导出导入 | 不同 Tab 的 DOM 结构不同 |
| P8 | 数据结构 | 保留层级结构 | 列表平铺会丢失信息 |
| P9 | 导入预览 | 全部可编辑，标记来源 | 用户可按需调整 |
| P10 | 模式切换 | 自动检测 + 手动覆盖 | 检测页面语言特征推荐模式 |
| P11 | 字段 key | name/id 为主 key + label | 保证稳定性 + 便于识别 |
| P12 | 搜索下拉框导出 | {text, id, fieldType} | 文本用于搜索，ID 留作参考 |
| P13 | 搜索下拉框填写失败 | 标记跳过，不阻塞流程 | 搜索框字段占比不高 |
| P14 | 页面区分 | 自动检测 + 按钮隔离 | 只注入当前模式对应的按钮 |
| P15 | 大模型翻译 | 一次请求打包全部自由文本 | 减少调用次数，保持上下文 |
| P16 | 面板形态 | 页面内浮层 | 不依赖新版 Chrome API |
| P17 | 按钮位置 | 页面底部浮动 | 跨页面统一体验 |
| P18 | 转换规则 | JSON 配置 + JS 函数 | 数据映射用 JSON，格式转换用 JS |
| P19 | Popup 范围 | 设置 + 状态概览 | 模式切换、API 配置、最近状态 |
| P20 | 验收方式 | 半自动验证 | 自动回读对比 + 人工保存 |

## 四、用户流程

### 4.1 导出

```
国内版产品编辑页 → 点击浮动"导出"按钮
  → 提取当前 Tab 表单数据 → 生成层级 JSON → 复制到剪贴板
  → 剪贴板失败时降级为 JSON 文件下载
```

### 4.2 导入

导入面板有 5 个状态：`input → preview → match → filling → verify`。

```
国际版产品编辑页 → 点击浮动"导入"按钮 → 弹出浮层面板
  → [input] 粘贴 JSON（或上传文件/选模板）
  → 点击"转换"
  → [preview] 转换预览：
      • 自动转换：null→空字符串、简体→繁体（语言字段）、预置规则映射
      • 全部字段可编辑，标记来源（auto-converted / ai-translated）
      • 每个字段有 3 个复选框：忽略 / AI 改写
      • 用户可勾选需要 AI 改写的字段 → 点击"AI 改写" → 弹出 AIRewriteDialog
  → 点击"匹配预览"
  → [match] 匹配预览：
      • 提取当前页面字段映射（extractFieldMap）
      • 按字段 label 匹配导入数据
      • 在页面控件上叠加预览 label（可视化）
      • 面板显示：未匹配字段（默认展开，带"选择控件"按钮）+ 已匹配字段（折叠，可"重新选择"）
      • 用户可手动绑定未匹配字段到页面控件
  → 点击"确认填写"
  → [filling] 逐字段填写：
      • FormFiller._fillField() 按字段类型分发
      • 单控件走智能匹配（FieldMatcher）→ antd1-filler → react-filler 兜底
      • 复合行走 label[for] 行级定位
      • 专用复合（recommendReason/richText）走专用方法
      • 进度回调更新 UI
  → [verify] 回读验证：
      • 重新提取页面表单数据
      • 与导入数据逐字段对比
      • 生成验证报告（成功/失败/跳过）
      • 用户人工确认后保存
```

### 4.3 AI 改写流程

```
转换预览面板 → 用户勾选字段的"AI"复选框 → 点击"AI 改写"按钮
  → AIRewriteDialog.show() 弹出对话框
  → 用户选择目标语言（英文/繁体）
  → 通过 background/service-worker.js 代理调用 LLM API（绕过 CORS）
  • prompt 包含：字段内容 + 目标语言 + 国际版字符清洗规则
  • 本地 _sanitizeForInternationalRules() 二次清洗（仅允许中文/英文/数字及指定符号）
  → 返回改写结果 → 写回预览面板对应输入框
```

### 4.4 模板系统流程

```
【保存模板】
  转换预览面板 → 调整字段值/忽略/AI 设置 → 点击"保存为模板"
  → 输入模板名称 → 保存到 chrome.storage.local
  • 模板内容：字段值 + 忽略配置 + AI 启用配置 + 默认值
  • 模板键：按页面 URL + Tab 区分
  • 最多 25 个模板

【应用模板】
  导入面板 [input] 状态 → 选择"使用已保存的模板" → 下拉选择模板
  → 模板配置合并到当前导入数据（覆盖字段值、应用忽略/默认值）

【管理模板】
  页面底部"模板管理"按钮 → 弹出管理面板
  → 预览 / 编辑名称描述 / 删除
```

### 4.5 完整迁移（单产品）

```
国内版基础信息 Tab → 导出
  → 国际版基础信息 Tab → 导入 → 确认
  → 国内版行程 Tab → 导出
  → 国际版行程 Tab → 导入 → 确认
  → ... 重复 → 最终人工保存
```

## 六、转换规则

### 预置规则（本地，不调 LLM）

| 规则 | 示例 | 方式 |
|------|------|------|
| 直接复制 | 行程天数、人数限制等数值 | JSON 映射 |
| 格式转换 | 目的地：级联下拉 → 短横连接 | JS 函数 |
| 标题拼接 | `{Country} {N}D{N-1}N {TourType}` | JS 函数 |
| 固定映射 | 服务语言、币种枚举 | JSON 映射 |
| 保留模板 | bookingControls 等配置 | JSON 映射 |

规则文件位置：`rules/base-info.js` + `rules/field-mappings.json`。

### LLM 翻译（外部 API）

- 目标语言：中文 → 英文/繁体
- 仅翻译自由文本（产品名、描述、路线名），不翻译 ID/枚举
- 一次请求打包，返回结构化 JSON
- 失败回退为原文，标记"翻译失败"
- **国际版特有约束**：字符清洗规则（仅允许中文、英文、数字及指定符号）
- LLM 调用通过 `background/service-worker.js` 代理（绕过 CORS）

## 八、已知风险

| 风险 | 缓解 |
|------|------|
| DOM 结构变化导致选择器失效 | 语义化选择器 + 抽取常量 |
| React fiber API 变化 | fiber 仅为辅助，DOM 为主 |
| 搜索下拉框异步延迟 | 轮询等待 + 超时跳过 |
| LLM 翻译质量 | 预览可编辑 + 术语预置 |
| URL 相同无法区分国内外 | 语言特征检测 + 手动切换 |
