提前预订控件测试页面

用于测试插件能否正确提取提前预订字段（只提取控件数据，跳过说明性文本）

预订设置
(UTC+08:00)北京时间
天 前可订
假设今天为6月1号，设置提前2天，18：00前可订，则表示客人在今天18:00前可以预订到6月3号出发的产品，在今天18:00后能订到6月4号出发的产品。若客人预订时的时区和设置时区不同，系统会自动转换。由携程大数据统计，客户下单时间高峰时间为晚上，建议设置较晚预订时间点，如22:00。

预期导出结果：

{
  "提前预订": {
    "domKey": "bookingControls.advanceBooking",
    "label": "提前预订",
    "fieldType": "mixedGroup",
    "value": {
      "parts": [
        { "type": "inputNumber", "value": 2 },
        { "type": "separator", "text": "天" },
        { "type": "timePicker", "value": "18:00" },
        { "type": "separator", "text": "前可订" }
      ]
    }
  }
}

关键点：

  • ✅ 隐藏的时区选择器被跳过
  • ✅ 只提取了 4 个部分：inputNumber、separator(天)、timePicker、separator(前可订)
  • ✅ publicTip 说明文本被跳过
  • ❌ 不应该包含："假设今天为6月1号..." 的长文本
